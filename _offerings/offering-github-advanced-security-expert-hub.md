### Overview

Through this advisory service, GitHub’s experts will facilitate the creation of a dynamic and centralized hub. 

This platform will empower your users and more. The expert hub will include some of these key elements: on-demand learning opportunities (e.g., learning paths, webinar recordings, YouTube videos, blog posts, guides, and documentation), foster collaborative discussions around security challenges and solutions, and serve as a knowledge base for valuable security resources and insights, ultimately enhancing your organization's security culture.

### Topics

- On-demand learning resources  
- Expectations for developers  
- Innersourcing solutions  
- Soliciting feedback  
- Encouraging collaboration

### Customer benefits

The offering will help customers:

- **Improve operational efficiency**: A centralized platform streamlines access to resources and discussions, saving time and reducing confusion.  
- **Enhance security posture**: Knowledge sharing and community building can lead to faster identification and resolution of security issues, reducing risk.  
- **Increase collaboration and innovation**: A forum for collaboration fosters knowledge exchange, potentially leading to innovative solutions and best practices.  
- **Strengthen security culture**: By promoting learning and discussion, the program can strengthen the overall security culture within the organization.

### Goals

After completing this service, customers will: 

- **Centralize GHAS resources**: A single repository for all GHAS-related discussions, resources, and insights.  
- **Foster knowledge sharing**: Use this new platform for sharing best practices, tips, and solutions.  
- **Build a community**: The Expert Hub gives users a space to collaborate, discuss, and learn from each other.  
- **Communicate better internally**: This new channel helps privately share proprietary information and communicate with users.

### Audience

**Required**:

- GitHub admin(s)  
- Application security team

**Optional**:

- CI/CD leads  
- Engineering leads  
- Security champions

### Delivery details

- **Level:** Fundamentrals \[100\]  
- **Offering type:** Advisory service  
- **Format:** Remote  
- **Class size:** 20 participant maximum to facilitate discussion  
- **Schedule:** 4 days, 1 hour per day  
- **Customer pre-work:** Gather internal information about developer expectations (e.g. remediation timelines, exception process, innersource contributions etc.)

### Customer prerequisites

Before this training, the customer needs to have in place:

- GitHub Enterprise Cloud Account
